# 최종 프로젝트: End-to-End 데이터 파이프라인 구축

## 프로젝트 개요
Adobe Experience Platform을 활용하여 전체 데이터 파이프라인을 구축합니다. 웹사이트에서 데이터를 수집하여 프로필을 생성하고, 타겟 오디언스를 세그먼테이션한 후, 마케팅 채널로 활성화하는 완전한 워크플로우를 구현합니다.

## 프로젝트 목표
1. 웹/앱에서 고객 데이터 수집
2. 프로필 생성 및 통합
3. 타겟 오디언스 세그먼테이션
4. 마케팅 채널로 활성화
5. 성과 측정 및 최적화

## 요구사항

### 비즈니스 시나리오
온라인 리테일 회사 "Luma"의 고객 여정 전반을 관리합니다.

### 데이터 소스
- 웹사이트 활동 (페이지뷰, 장바구니, 구매)
- 고객 정보 (이름, 이메일, 선호도)
- 멤버십 정보 (가입일, 등급, 포인트)

## 구현 단계

### 1단계: 데이터 수집 설정

#### 스키마 생성
- [ ] Customer Profile Schema 생성
- [ ] Web Page View Event Schema 생성
- [ ] Commerce Event Schema 생성

#### Web SDK 구현
- [ ] HTML 페이지에 Web SDK 통합
- [ ] 페이지뷰 이벤트 전송
- [ ] 장바구니 추가 이벤트 전송
- [ ] 구매 이벤트 전송

#### 샘플 데이터 업로드
- [ ] 프로필 데이터 배치 수집
- [ ] 이벤트 데이터 검증

### 2단계: 프로필 및 Identity 관리

#### Identity 설정
- [ ] Identity Namespace 생성 (Email, Phone)
- [ ] 스키마에 Identity 필드 설정
- [ ] Identity Graph 확인

#### Merge Policy 생성
- [ ] Timestamp Ordering 정책 생성
- [ ] Priority Ordering 정책 생성 (필요 시)

#### 프로필 활성화
- [ ] 모든 스키마 프로필 활성화
- [ ] 프로필 샘플 확인

### 3단계: 세그멘테이션

#### 핵심 세그먼트 생성
세그먼트 1: 장바구니 이탈 고객
- [ ] 지난 7일 내 장바구니 추가
- [ ] 구매 없음

세그먼트 2: VIP 고객
- [ ] 총 구매 금액 $500 이상
- [ ] 또는 멤버십 포인트 1000 이상

세그먼트 3: 신규 고객
- [ ] 가입일 30일 이내
- [ ] 최소 1회 방문

세그먼트 4: 비활성 고객
- [ ] 마지막 방문 90일 이상 경과
- [ ] 구매 이력 있음

세그먼트 5: 재구매 고객
- [ ] 지난 30일 내 구매
- [ ] 이전 구매 이력 있음

#### 세그먼트 평가
- [ ] 배치 세그먼테이션 스케줄 설정
- [ ] 스트리밍 세그먼테이션 설정 (필요 시)
- [ ] 세그먼트 집계 확인

### 4단계: 목적지 및 활성화

#### 목적지 설정
목적지 1: Google Ads
- [ ] 연결 설정
- [ ] 세그먼트 활성화
- [ ] 데이터 흐름 모니터링

목적지 2: 이메일 플랫폼
- [ ] 연결 설정 (Mailchimp 또는 유사 플랫폼)
- [ ] 프로필 데이터 매핑
- [ ] 활성화 설정

목적지 3: 커스텀 HTTP
- [ ] HTTP API 목적지 생성
- [ ] 웹훅 엔드포인트 설정
- [ ] 데이터 형식 설정

#### 활성화 테스트
- [ ] 목적지로 데이터 전송 확인
- [ ] 외부 시스템에서 데이터 수신 확인
- [ ] 활성화 로그 확인

### 5단계: 모니터링 및 분석

#### Query Service 분석
- [ ] 세그먼트별 프로필 수 조회
- [ ] 세그먼트 간 중복 분석
- [ ] 활성화 상태 모니터링

#### 대시보드 설정
- [ ] 주요 메트릭 정의
- [ ] 일일 보고서 생성 (선택)
