## Workspace 기본 구조

### 패널 (Panel)

프로젝트의 최상위 컨테이너. 각 패널은 독립적인 날짜 범위와 세그먼트 적용 가능.

| 패널 유형 | 용도 |
|----------|------|
| 빈 패널 | 자유로운 분석 구성 |
| Attribution | 채널 귀속 분석 |
| Analytics for Target (A4T) | Target 실험 분석 |
| 세그먼트 비교 | 두 세그먼트 비교 |
| Quick Insights | 빠른 데이터 탐색 |

### 테이블 구성

```
┌─────────────────────────────────────────────────────┐
│ 차원 (Dimension)    │ 지표1  │ 지표2  │ 지표3     │
├─────────────────────────────────────────────────────┤
│ 값 1                │ 12,500 │ 35,000 │ 8,500     │
│ 값 2                │ 8,200  │ 22,000 │ 5,600     │
│ 값 3                │ 6,100  │ 18,000 │ 4,200     │
├─────────────────────────────────────────────────────┤
│ 합계                │ 26,800 │ 75,000 │ 18,300    │
└─────────────────────────────────────────────────────┘

- 행(Row): 차원 값
- 열(Column): 지표
- 드래그 앤 드롭으로 구성
```

## 시각화 유형

### 추세 분석

**라인 차트**
- 시간 경과에 따른 추세 파악
- 복수 지표 비교
- 예: 일별 방문/페이지뷰 추이

**영역 차트**
- 누적 값 시각화
- 전체 대비 비중 파악
- 예: 채널별 트래픽 누적

### 비교 분석

**막대 차트 (가로/세로)**
- 카테고리 간 비교
- 예: 기기별 방문 수

**글머리 기호 그래프**
- 목표 대비 실적
- 예: KPI 달성률

### 분포 분석

**히스토그램**
- 데이터 분포 확인
- 예: 주문 금액대별 분포

**산점도**
- 두 변수 간 상관관계
- 예: 체류 시간 vs 전환율

### 구성 분석

**도넛/파이 차트**
- 전체 대비 비중
- 예: 기기 유형별 점유율

**트리맵**
- 계층적 비중 표현
- 예: 카테고리별 매출 비중

### 흐름 분석

**폴아웃 (Fallout)**
- 단계별 이탈률
- 전환 퍼널 분석
- 순차적 vs 비순차적 옵션

**플로우 (Flow)**
- 사용자 경로 시각화
- 진입/이탈 페이지 분석
- 양방향 탐색

**코호트 테이블**
- 사용자 그룹별 행동 추적
- 재방문율, 유지율 분석

### 기타 시각화

**요약 숫자/변화**
- 핵심 KPI 표시
- 이전 기간 대비 변화율

**벤 다이어그램**
- 세그먼트 간 중복
- 최대 3개 세그먼트

**지도**
- 지역별 데이터
- 국가/지역 코드 필요

## 리포트 템플릿

### 1. 트래픽 개요 대시보드

```
┌─────────────────────────────────────────────────────┐
│ 패널: 트래픽 개요                                   │
│ 날짜: 지난 30일                                     │
├─────────────────────────────────────────────────────┤
│ [요약 숫자]      [요약 숫자]      [요약 숫자]       │
│  방문: 125,000   방문자: 85,000   페이지뷰: 350K   │
│  +12% ▲         +8% ▲           +15% ▲            │
├─────────────────────────────────────────────────────┤
│ [라인 차트] 일별 방문 추이                          │
│ - 방문                                              │
│ - 고유 방문자                                       │
├─────────────────────────────────────────────────────┤
│ [테이블] 트래픽 소스별                              │
│ 마케팅 채널 | 방문 | 방문자 | 이탈률               │
├─────────────────────────────────────────────────────┤
│ [도넛] 기기별 비중    │ [막대] 상위 페이지          │
└─────────────────────────────────────────────────────┘
```

**구성 요소:**
- 차원: 일(Day), 마케팅 채널, 기기 유형, 페이지
- 지표: 방문, 고유 방문자, 페이지뷰, 이탈률
- 날짜 비교: 이전 30일

### 2. 전환 퍼널 분석

```
┌─────────────────────────────────────────────────────┐
│ 패널: 전환 퍼널                                     │
├─────────────────────────────────────────────────────┤
│ [폴아웃 시각화]                                     │
│                                                     │
│ 홈페이지 ─────► 상품목록 ─────► 상품상세            │
│ 100%           75%              52%                 │
│           -25%            -23%                      │
│                                                     │
│ 상품상세 ─────► 장바구니 ─────► 결제완료            │
│ 52%            28%              12%                 │
│           -24%            -16%                      │
├─────────────────────────────────────────────────────┤
│ [테이블] 단계별 상세                                │
│ 단계        | 진입     | 이탈     | 전환율          │
│ 홈페이지    | 100,000  | -        | 100%            │
│ 상품목록    | 75,000   | 25,000   | 75%             │
│ 상품상세    | 52,000   | 23,000   | 69%             │
│ 장바구니    | 28,000   | 24,000   | 54%             │
│ 결제완료    | 12,000   | 16,000   | 43%             │
├─────────────────────────────────────────────────────┤
│ [막대] 이탈 페이지 분석 (장바구니 이탈자)           │
└─────────────────────────────────────────────────────┘
```

**구성 요소:**
- 폴아웃 터치포인트: 페이지 또는 이벤트
- 세그먼트: 전체, 신규, 재방문자 비교
- 분석: 단계별 드롭오프, 이탈 사유

### 3. 캠페인 성과 분석

```
┌─────────────────────────────────────────────────────┐
│ 패널: 캠페인 성과                                   │
│ 세그먼트: 마케팅 캠페인 유입                        │
├─────────────────────────────────────────────────────┤
│ [테이블] 캠페인별 성과                              │
│ 추적코드   | 방문   | 주문  | 매출      | ROI       │
│ CAMP001   | 15,000 | 450  | ₩45,000,000 | 320%     │
│ CAMP002   | 12,000 | 280  | ₩28,000,000 | 180%     │
│ CAMP003   | 8,000  | 120  | ₩12,000,000 | 85%      │
├─────────────────────────────────────────────────────┤
│ [라인] 캠페인 기간 추이                             │
├─────────────────────────────────────────────────────┤
│ [Attribution 패널]                                  │
│ 모델: 첫 터치 / 마지막 터치 / 선형 비교             │
└─────────────────────────────────────────────────────┘
```

**구성 요소:**
- 차원: 추적 코드 (eVar)
- 지표: 방문, 주문, 매출, ROI (계산된 지표)
- 귀속 모델 비교

## 계산된 지표

### 기본 계산 지표

```
// 전환율
전환율 = 주문 / 방문 × 100
유형: 백분율, 소수점 2자리

// 평균 주문 금액 (AOV)
AOV = 매출 / 주문
유형: 통화, 소수점 0자리

// 평균 페이지뷰
세션당 페이지뷰 = 페이지뷰 / 방문
유형: 숫자, 소수점 1자리

// 이탈률
이탈률 = 이탈 / 진입 × 100
유형: 백분율

// 장바구니 전환율
장바구니 전환율 = 주문 / 장바구니 추가 × 100
유형: 백분율
```

### 고급 계산 지표

```
// 장바구니 이탈률
장바구니 이탈률 = (장바구니 추가 - 주문) / 장바구니 추가 × 100

// 고객 생애 가치 (간이)
CLV = 평균 주문 금액 × 평균 구매 빈도

// ROI
ROI = (매출 - 광고비) / 광고비 × 100

// 상품 전환율
상품 전환율 = 구매 / 상품 조회 × 100

// 검색 성공률
검색 성공률 = 검색 후 클릭 / 검색 × 100
```

### 세그먼트 적용 지표

```
// 신규 방문자 전환율
[세그먼트: 신규 방문자] 주문 / [세그먼트: 신규 방문자] 방문 × 100

// 모바일 매출 비중
[세그먼트: 모바일] 매출 / 총 매출 × 100

// VIP 평균 주문 금액
[세그먼트: VIP 회원] 매출 / [세그먼트: VIP 회원] 주문
```

## 세그먼트 활용

### 기본 세그먼트 유형

**방문자 세그먼트 (Visitor)**
```
조건: 구매 이벤트 발생
결과: 한 번이라도 구매한 모든 방문자의 전체 행동 포함
```

**방문 세그먼트 (Visit)**
```
조건: 구매 이벤트 발생
결과: 구매가 발생한 방문 세션만 포함
```

**히트 세그먼트 (Hit)**
```
조건: 구매 이벤트 발생
결과: 구매 이벤트가 발생한 히트만 포함
```

### 주요 세그먼트 예시

**신규 vs 재방문자**
```
신규: 방문 횟수 = 1
재방문: 방문 횟수 > 1
```

**구매자 vs 비구매자**
```
구매자: 주문 이벤트 존재 (방문자 레벨)
비구매자: 주문 이벤트 존재하지 않음
```

**장바구니 이탈자**
```
조건1: 장바구니 추가 이벤트 존재
AND
조건2: 주문 이벤트 존재하지 않음
컨테이너: 방문 레벨
```

**고가치 고객**
```
조건: 매출 >= ₩1,000,000 (방문자 레벨)
또는
조건: 주문 횟수 >= 5 (방문자 레벨)
```

**모바일 구매자**
```
조건1: 모바일 기기 유형 = '모바일'
AND
조건2: 주문 이벤트 존재
```

### 순차적 세그먼트

```
// 특정 경로를 따른 사용자
페이지 A 본 후 → 페이지 B 본 후 → 구매
(THEN 연산자 사용)

// 시간 제약 추가
장바구니 추가 후 → 24시간 내 → 구매
(Within 연산자 사용)
```

## 고급 분석

### Attribution IQ (귀속 분석)

**귀속 모델:**

| 모델 | 설명 |
|------|------|
| 첫 번째 터치 | 첫 접점에 100% 귀속 |
| 마지막 터치 | 마지막 접점에 100% 귀속 |
| 선형 | 모든 접점에 균등 분배 |
| 시간 감소 | 최근 접점에 더 많이 귀속 |
| U자형 | 첫/마지막에 40%, 중간에 20% |
| J자형 | 마지막에 60%, 첫 번째에 20%, 중간에 20% |
| 역J자형 | 첫 번째에 60%, 마지막에 20%, 중간에 20% |
| 알고리즘 | ML 기반 자동 귀속 |

**적용 방법:**
1. 지표 선택 → 톱니바퀴 아이콘
2. '기본이 아닌 귀속 모델 사용' 체크
3. 모델 및 전환 확인 기간 선택

### 이상 탐지 (Anomaly Detection)

자동으로 비정상 패턴 감지:
- 일별 데이터에서 예상 범위 벗어난 값 표시
- 계절성, 추세 고려한 통계 모델 사용
- 라인 차트에서 자동 하이라이트

### 기여도 분석 (Contribution Analysis)

이상치 발생 원인 자동 분석:
1. 이상 탐지된 포인트 클릭
2. '기여도 분석 실행'
3. 상위 기여 차원 값 확인

### 세그먼트 비교

두 세그먼트 간 통계적 차이 분석:
1. 세그먼트 비교 패널 추가
2. 두 세그먼트 드래그
3. 자동으로 유의미한 차이점 표시

### 코호트 분석

**유지율 분석:**
```
포함 기준: 첫 구매
반환 기준: 재구매
세분화: 주/월
```

**맞춤 코호트:**
```
포함 기준: 특정 캠페인 유입
반환 기준: 사이트 재방문
기간: 12주
```

## 팁과 모범 사례

### 성능 최적화

1. **날짜 범위 제한**: 필요한 기간만 조회
2. **차원 값 제한**: settings.limit 사용
3. **필요한 지표만 포함**: 불필요한 지표 제거
4. **세그먼트 단순화**: 복잡한 중첩 피하기

### 리포트 공유

- **프로젝트 공유**: 권한 설정 (보기/편집)
- **PDF 내보내기**: 정적 리포트 배포
- **예약 전송**: 정기 리포트 자동화
- **큐레이션**: 필요한 구성요소만 표시

### 명명 규칙

```
프로젝트: [팀명]_[분석유형]_[기간]
예: 마케팅_캠페인성과_2024Q1

세그먼트: [범위]_[조건]_[설명]
예: 방문자_구매_VIP고객

계산지표: [카테고리]_[지표명]
예: 전환_장바구니전환율
```
